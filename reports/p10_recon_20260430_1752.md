## 1.3 — 訊號路徑等價性 diff（live `check_signal` vs backtest `run_backtest_masr`）

對 `MaSrBreakoutStrategy.check_signal` (live) 跟 `run_backtest_masr` (backtest) 的訊號邏輯逐項 diff：

### A. 進場價計算 — ✅ 一致

| | Live | Backtest |
|---|------|----------|
| 取 K 線 | `df = self._get_klines(symbol, '4h', limit=130)` | `df_tf = fetch_klines(client, symbol, '4h', months)` |
| 確認已收盤的 bar | `df_a = df.iloc[:-1]` 然後 `latest = df_a.iloc[-1]` (倒數第二根) | 主迴圈 `for i in range(warmup, len(df_tf) - 1):` 用 `df_tf.iloc[i]` |
| `cur_close` | `float(latest["close"])` | `float(df_tf["close"].iloc[i])` |

兩者都用「已收盤的 bar 的 close」作為 entry price。Live 是處理「最新一根 K 線收盤事件」當下；backtest 對歷史每根都跑一次。**邏輯等價**。

### B. SL / TP 計算公式 — ✅ 完全一致

Both:
```python
sl_atr = cur_close - MASR_SL_ATR_MULT * atr_v
sl     = max(sl_atr, ema50_v)         # 取較緊的
sl_dist = cur_close - sl
tp1    = cur_close + MASR_TP1_RR * sl_dist
tp2    = cur_close + MASR_TP2_RR * sl_dist
```

逐字一致。**TP1 後 SL 移到保本** (`MASR_BE_AFTER_TP1`) — 在 backtest 由 `simulate_trade(sl_to_be_after_tp1=True)` 模擬；在 live 由 `Signal.use_trailing=True + executor` 處理（`use_trailing` 觸發 ATR trailing 而非死式 BE，**這是已知不對稱**：backtest 用 fixed BE 模擬，live 用 ATR trailing）。

### C. ATR / S-R lookback — ✅ 同 series 同 window

| | Live | Backtest |
|---|------|----------|
| ATR series | `ta.atr(df_a["high/low/close"], length=14)` | `ta.atr(df_tf["high/low/close"], length=14)` |
| ATR percentile filter | `atr_recent = atr_s.iloc[-MASR_RES_LOOKBACK:]` | `atr_window = atr_s.iloc[i - lookback + 1 : i + 1]` |
| Resistance lookback | `df["high"].iloc[-lookback:].values` | `highs_arr[end_idx - lookback : end_idx]` |
| Resistance algorithm | `_find_active_resistance(self, df, atr)` (cluster + min_touches + max breakable) | `_bt_masr_find_resistance(highs_arr, end_idx, atr, ...)` (相同 cluster 演算法) |

兩者用同 series、同 window 大小、相同 cluster + max-breakable 邏輯。**等價**。

### D. Module-level Config 讀取（會鎖死 .env）— ✅ 沒有問題

| | Module-level Config reads |
|---|---|
| `scripts/backtest.py` | `BT_FIB_TOL`, `BT_VOL_MULT`, `BT_SKIP_VOL_RISE` (僅 NKF 用，line 61-63) |
| `scripts/strategies/ma_sr_breakout.py` | **無** — 全部 `from config import Config` 都在方法內 |

MASR 路徑沒有 module-level Config rebind。改 .env 後重啟生效，沒有 hot-reload 但也沒有 import-time freeze 風險。

⚠️ 但 `Config` class 本身是在第一次 `import config` 時讀 `os.getenv` 算出 class attrs。**process 啟動後不會重讀 .env**——這對 backtest sweep 用的 `ConfigPatch` 機制是必要前提，對 live 也意味著**改 .env 必須重啟 bot 才生效**。

### E. 邏輯差異點（非 bug，但要記錄）

以下是 live 跟 backtest 不完全等價的地方，shadow comparison 會誤報的「正常 diff」：

1. **30 日漲幅過濾**（中等差異）
   - Live: 在 `screen_coins()` 用日線算 30d return ≥ `MASR_MIN_30D_RETURN_PCT`，**選幣時一次過濾**
   - Backtest: 在主迴圈內用 4h × 180 bars 算 30d return（180×4h ≈ 30 天，對齊但不完全等價，因為 4h 跟 1d 的 bar 邊界不同）
   - 影響：backtest 可能在某根 4h 拒絕的 bar，live 因為是日線檢查而留下；或反之
   - **shadow 應允許 30d return tier 偏移 ±2%**

2. **TP1 後 SL 移動**（已知不對稱）
   - Live: `Signal.use_trailing=True` → executor 用 ATR trailing
   - Backtest: `sl_to_be_after_tp1=True` → fixed move-to-breakeven
   - 影響：TP2 後的退場價 / 是否打 TP2 可能不同；**只影響 trade outcome 不影響進場 signal**

3. **ADX 評分**
   - Live: `_score_signal` 計算 ADX > 30 加分
   - Backtest: 評分區塊註解「ADX bonus 在回測中跳過」
   - 影響：兩邊的 score 可能差 1（live 多算 ADX bonus）。當 `min_score=2` 時可能造成 live 通過、backtest 拒絕（或反之）的 boundary case
   - **shadow score 容差 ±1**

4. **`MASR_MIN_BREAKOUT_PCT` 預設值**（無實質影響）
   - Live: `getattr(Config, "MASR_MIN_BREAKOUT_PCT", 0.005)`（fallback 0.5%）
   - Backtest: `float(Config.MASR_MIN_BREAKOUT_PCT)` (直接讀，default 0.0)
   - Config 本身定義 default 0.0 → `getattr` 取到 0.0 → fallback 0.005 永遠不觸發。**現況等價但是 code smell**——live 的 fallback 是死字，建議統一寫法（不在這輪改）。

5. **`atr_window` 起點偏 1 根**（極微差，已測過 P4 audit 證明影響可忽略）
   - Live: `atr_s.iloc[-MASR_RES_LOOKBACK:]` （含當前 bar 的 atr）
   - Backtest: `atr_s.iloc[i - lookback + 1 : i + 1]` （也含當前 bar 的 atr）
   - 仔細看是等價的（都是 lookback 個 element 含當前）

### F. Shadow comparison 設計建議（給 Phase 3）

容差設定：
- `direction (LONG/SHORT)` — 必須完全一致，不容差
- `entry_price` — 容差 < 0.05%（應該 0%，加 0.05% 是浮點 + tick rounding 餘裕）
- `sl / tp1 / tp2` — 容差 < 0.05%
- `score` — 容差 ±1（因 ADX bonus）
- 缺一邊有訊號、另一邊沒訊號 → 必須 alert（不容差）

時序對齊：
- live 觸發是「4h K 線收盤後的事件」 → 用該 K 線收盤時刻的 timestamp
- backtest 用 `df_tf.iloc[i]` 對應同一 timestamp 的 bar
- shadow_compare 用 `df_klines_4h` 切到 `bar_time` 為止，再呼叫 `MaSrBreakoutStrategy.check_signal` 等價的 backtest path 重跑
